# 四、详细过程说明（以 Stage 1.5 实验为例）

本周重点推进了 Stage 1.5 翻译条件下的 hidden state 对齐实验。

实验过程中：

1. 首先构建翻译数据集；
2. 使用 target token hidden state 进行 MSE 对齐；
3. 对不同 adapter 结构进行实验；
4. 统计 validation loss 与相似度指标；
5. 分析不同层接口的稳定性。

在实验初期，loss 数值较大，怀疑存在 hidden size 不一致导致的信息压缩问题。随后调整 adapter 结构，并尝试更合适的接口组合后，实验结果逐渐稳定。

最终发现：

* 17 → 16 接口在跨模型实验中表现较好；
* CKA 与 cosine similarity 有明显提升；
* 翻译条件下的表示对齐比单纯 hidden 对齐更有效。

# 五、遇到的问题及解决方法

## 1. 跨模型拼接效果不稳定

问题：

不同模型 hidden state 分布差异较大，直接拼接容易出现输出异常。

解决方法：

* 增加 adapter 对齐层；
* 调整接口层位置；
* 引入翻译条件下的对齐训练。

---

## 2. loss 数值过大

问题：

高维 hidden state 映射时 raw MSE 数值较高。

解决方法：

* 对 loss 做 mean normalization；
* 分析 hidden size 差异；
* 区分“loss 数值大”与“训练是否有效”两个问题。
